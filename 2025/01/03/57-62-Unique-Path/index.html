56-62. Unique Path

發表於 2025-01-03 分類於 grind75

題目描述

There is a robot on an m x n grid. The robot is initially located at the top-left corner (i.e., grid[0][0]). The robot tries to move to the bottom-right corner (i.e., grid[m - 1][n - 1]). The robot can only move either down or right at any point in time.

Given the two integers m and n, return the number of possible unique paths that the robot can take to reach the bottom-right corner.

The test cases are generated so that the answer will be less than or equal to 2 * 109.

Example 2:

Constraints:

  • 1 <= m, n <= 100

想法

這題給了一個m*n matrix，希望求出從座標(0, 0)到(n-1, m-1)有多少unique path數。

如果以top-down的角度來看，可以定義traverse(i, j)為從(0, 0)到(i, j)的unique path數，而我們每次只能選擇往右走一步或往下走一步，只要把往右走一步和往下走一步的unique path數加總即為所求。

解法

1. Recursion

Time complexity: O(2^(m+n))

  • Recursion tree的深度為m+n (會traverse到(n-1, m-1))，而對每個座標都有往右走一步或往下走一步的選擇，因此時間複雜度大約為O(2^(m+n))。

Space complexity: O(m+n)

  • Recursion tree的深度為m+n

2. Recursion + Memo

Time complexity: O(m*n)

  • matrix中所有座標的unique path都被紀錄在memo中，而memo的size最大為m*n

Space complexity: O(m*n)

  • memo的size最大為m*n

3. DP

進一步用DP來解決這題(bottom-up視角)，定義dp[row][col]為(0, 0)到(n-1, m-1)的unique path數。

State transition:

  • dp[row][col] = dp[row-1][col] + dp[row][col-1]

Base cases:

  • dp[0][col] = 0, where 0 <= col <= m-1

  • dp[row][0] = 0, where 0 <= row <= n-1

  • dp[1][1] = 1

Time complexity: O(m*n)

  • traverse所有座標，座標數為m*n。

Space complexity: O(m*n)

  • dp大小。
